## 작업 상세 내용

### 1. "새 여행 시작 (사진 다중 업로드)" UI 구현 (표지/목차 화면)
- 기존의 수동 입력 기반 "새 여행 추가(FAB 버튼 등)" 모달을 다중 사진 업로드 존(Drag & Drop 지원)으로 개편하라.
- 사용자가 여러 장의 사진을 선택하면 `POST /api/trips/from-photos` 엔드포인트(멀티파트 폼 데이터: `photos` 필드)로 전송하라.
- 전송 및 AI 분석이 진행되는 동안 화면 전체에 감성적인 **로딩 오버레이**를 띄워라 (예: "AI가 사진을 분석하여 여행의 조각들을 맞추고 있습니다...").

### 2. 자동 생성된 여행 페이지 진입 및 제목 수정(Inline Edit)
- 응답(`{ success: true, data: { trip, photos, summary } }`)을 받으면, `state.trips`에 새 여행을 추가하고, 화면을 해당 여행의 상세 페이지(내지)로 전환하라.
- **제목 인라인 편집**: 상세 페이지의 헤더에 표시되는 제목(`trip.title`)을 클릭하면 즉시 `<input>`으로 바뀌어 수정할 수 있는 기능을 구현하라.
  - 수정 완료 시(엔터 키 또는 포커스 아웃), `PATCH /api/trips/:id` 를 호출하여 서버(`{ title: '새 제목' }`)에 업데이트하고 화면에 반영하라. (이 엔드포인트는 방금 추가됨)

### 3. 타임라인(연대기) 정렬 및 태그 표시
- 여행 상세 페이지에 사진을 렌더링할 때, `taken_at` 기준 **시간 순서대로 정렬**하여 세로 스크롤 타임라인 형태로 표시하라.
- 각 사진 컴포넌트 하단이나 오버레이에 `vision_tags` (예: `time_of_day`, `environment`)를 작고 예쁜 뱃지(태그) 형태로 표시하라.

### 4. 이 여행의 "미분류 사진 서랍" UI 추가
- 응답으로 받은 사진 중 `classified === false` 인 사진들을 일반 타임라인과 분리하여, 해당 여행 상세 페이지의 우측 하단이나 별도의 **서랍/토글 영역**에 표시하라.
- (드래그 앤 드롭으로 타임라인 이동하는 기능은 차후 고도화 예정이므로, 현재는 시각적으로 "서랍 공간"에 분리하여 보여주는 데 집중하라.)
