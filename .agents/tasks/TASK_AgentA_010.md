## 작업 목적
백엔드의 새 API(삭제 기능, 기간 자동 생성)를 프론트엔드에 연동하고, 제목 아래에 표시되는 날짜(기간) 정보도 제목처럼 클릭하여 수동으로 수정할 수 있도록 인라인 편집 기능을 구현한다.

---

## 작업 상세 내용

### 1. 여행 및 사진 삭제 UI 및 API 연동
- **여행 삭제**:
  - 목차(표지) 화면의 각 여행 카드 또는 여행 상세 페이지 우측 상단 등에 `[삭제]` 버튼을 추가하라.
  - 클릭 시 `window.confirm("정말로 이 여행과 모든 사진을 삭제하시겠습니까? 복구할 수 없습니다.")` 으로 경고 모달을 띄워라.
  - 확인 시 `DELETE /api/trips/:id` 를 호출하고, 성공하면 `state.trips`에서 해당 여행을 제거한 뒤 표지 화면으로 이동하라.
- **사진 개별 삭제**:
  - 여행 상세 페이지의 타임라인 및 미분류 서랍 내 각 사진 항목에 `[휴지통]` 모양의 삭제 아이콘을 추가하라.
  - 클릭 시 `DELETE /api/photos/:id` 를 호출하고, 성공하면 현재 화면의 렌더링 리스트(`state.photos` 등)에서 해당 사진을 즉시 제거하라.

### 2. 제목(Title) 수정 API 연동 점검
- 이전에 구현한 인라인 수정 기능이 `PATCH /api/trips/:id` 엔드포인트를 호출하여 `{ title: '새 제목' }` 형태로 서버에 잘 저장되는지 확인하고, 필요하다면 연동하라.

### 3. 날짜/기간(Description) 인라인 수정 기능 구현
- 여행 제목 바로 아래에 작게 표시되던 `trip.description` (예: "6월 15일 ~ 6월 17일 (2박 3일)") 텍스트를 제목 수정 기능과 동일한 방식(인라인 에디터)으로 구현하라.
- 해당 날짜 텍스트를 클릭하면 `<input type="text">`로 변환되어 사용자가 자유롭게 문구를 수정할 수 있게 하라.
- 수정 후 엔터를 누르거나 포커스를 잃으면 `PATCH /api/trips/:id`를 호출하여 `{ description: '수정된 텍스트' }` 형태로 백엔드에 업데이트하고, 뷰를 갱신하라.
